# Mongtory Diary: Product Specification & Roadmap

This document outlines the vision, core features, and development roadmap for the Mongtory Diary project.

## 1. Vision & Goals
- **Mission**: To provide a joyful and emotional journaling experience with the "Mongtory" character.
- **Goal**: Create an engaging diary service that users look forward to using daily.
- **Core Value**: Emotional connection through character interaction and intuitive UX.

## 2. Core Features (Product Scope)

### 2.1. Diary Entry & Management
- **Daily Log**: Create, Read, Update, and Delete (CRUD) entries with text and photos.
- **Emotion Stickers**: Tag entries with "Mongtory" emotion stickers (Happy, Sad, Calm, etc.).
- **Calendar View**: Monthly view of entries for quick navigation.

### 2.2. Character Interaction (Phase 2+)
- **Emotional Feedback**: Mongtory's expressions and dialogues react to the user's entry sentiment.
- **Achievement System**: Unlock items or backgrounds based on writing frequency.

### 2.3. System Utility
- **Search & Filter**: Find entries by date range or tags.
- **User Sync**: Account-based synchronization across devices.

## 3. Technical Roadmap (Phases)

### Phase 1: MVP (Minimum Viable Product)
- User Authentication (Signup/Login).
- Basic Diary CRUD with SQLite persistence.
- Calendar-based navigation UI.

### Phase 2: Engagement & Rich Media
- Emotion sticker implementation.
- Image upload (local storage or cloud).
- Basic character reaction system.

### Phase 3: Analytics & Customization
- Sentiment analysis report (Weekly/Monthly).
- Theme customization (Character-based).

## 4. UI/UX Design Concept
- **Palette**: Warm, pastel colors for a cozy atmosphere.
- **Typography**: Cute yet readable fonts (e.g., Sans-serif mixed with handwritten style).
- **Interactions**: Fast page transitions using Vite/React and subtle micro-interactions for emotional feedback.
