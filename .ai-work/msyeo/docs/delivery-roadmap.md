# Mongtory Diary 단계적 업무계획

## 목적
Spring Boot 백엔드와 Flutter 앱을 중심으로 실제 사용 가능한 감정형 일기 MVP를 완성하기 위한 단계별 실행 계획이다. PM, QA, FE 개발자, BE 개발자는 이 문서를 목표 로드맵으로 보고, 실제 요청과 진행 기록은 `.ai-work/msyeo/docs/collaboration/` 아래 역할별 문서에 남긴다.

## 현재 기준선
- 백엔드: 회원가입, 로그인, 토큰 재발급, 일기 CRUD, 캘린더, 감정 목록 API 골격이 있다.
- Flutter: 로그인 입력 폼, 일기 목록/상세/생성/수정/삭제, 캘린더, 프로필, mock/remote 전환 구조가 있다.
- QA: widget test 하네스와 로그인 이후 주요 조회 API remote 회귀 검증 기록이 있다.
- 협업: PM/QA/FE/BE 역할별 문서와 handoff, responses, 룰 확인 기록 체계를 사용한다.

## 공통 주의
- 모든 담당자는 작업 시작 전 변경 룰 확인 기록을 자기 역할 `status.md`에 남긴다.
- 공통 인덱스는 PM이 갱신하고, 각 역할은 자기 역할 디렉토리 문서를 우선 갱신한다.
- 작업 블록 종료 시 역할별 `handoff/YYYY-MM-DD.md`를 작성한다.
- 커밋은 PM이 완료 요청 단위로 선별해 수행한다.
