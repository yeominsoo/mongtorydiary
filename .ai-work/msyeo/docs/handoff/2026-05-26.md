## 현재 동작 상태
- `git fetch --all --prune` 후 현재 로컬 `feature/ai-workspace-docs`는 `origin/feature/ai-workspace-docs`와 일치한다.
- GitOps/k3s 배포는 `origin/master` 기준이며, 현재 배포 이미지는 `ghcr.io/yeominsoo/mongtorydiary-backend:c6a90bbb6e189e24c2fdc58b8ea7a399950a7f37`, `ghcr.io/yeominsoo/mongtorydiary-frontend:c6a90bbb6e189e24c2fdc58b8ea7a399950a7f37`이다.
- k3s `mongtorydiary` namespace의 `frontend`, `backend`, `db` pod는 모두 `Running`이다.
- `http://127.0.0.1:30080/`는 nginx FE HTML 200을 반환한다.
- `http://127.0.0.1:30081/api/v1/emotions`는 Spring Boot API JSON 200을 반환한다.
- 현재 운영 웹 FE는 `mobile-flutter/`가 아니라 `frontend/` React 프로토타입이며, 화면에는 `Mongtory Diary`, `React + TypeScript + Redux + Spring Boot 3` 정도만 표시된다.

## 미해결 이슈
- 사용자가 기대한 Flutter 앱 화면은 현재 k3s 운영 웹으로 배포되어 있지 않다.
- `feature/ai-workspace-docs`와 `origin/master`는 동일 커밋 상태가 아니다. 실제 배포는 `origin/master`와 GitOps 이미지 태그를 기준으로 판단해야 한다.
- Flutter web을 운영 웹으로 제공하려면 `Dockerfile.frontend`를 `mobile-flutter` release build 기반으로 전환하거나, React web과 분리된 별도 Flutter web 배포 서비스를 추가해야 한다.
