# 캘린더와 몽토리 메뉴 제품 범위

## 목적
`REQ-20260521-04` 기준으로 캘린더와 몽토리 메뉴를 출품 가능한 MVP 화면으로 정의한다. 두 메뉴는 단순 보조 화면이 아니라 사용자가 매일 기록하고 돌아오게 만드는 핵심 흐름으로 둔다.

## 캘린더 메뉴
캘린더는 월간 기록과 실행 계획을 한 화면에서 확인하는 작업 공간이다.

### 제공 기능
- 큰 월간 달력으로 현재 월을 표시한다.
- 이전 달, 다음 달, 오늘 이동을 제공한다.
- 날짜 셀에는 일기 수와 TODO 완료 현황을 함께 표시한다.
- 날짜를 누르면 아래 상세 패널에서 해당 날짜의 일기와 TODO를 확인한다.
- 해당 날짜에 바로 일기를 작성할 수 있다.
- 해당 날짜에 TODO를 추가할 수 있다.
- TODO 완료 체크와 삭제를 지원한다.

### 데이터 기준
- 일기는 기존 `DiarySummary` 목록과 월간 캘린더 요약을 사용한다.
- TODO는 `GET /api/v1/todos?from=YYYY-MM-DD&to=YYYY-MM-DD`로 보이는 월 전체를 조회한다.
- 월간 캘린더 요약은 일기만 있는 날짜뿐 아니라 TODO만 있는 날짜도 활성 날짜로 내려준다.
- 빈 날짜 셀은 Flutter 화면에서 월 정보를 기준으로 만든다.

### MVP에서 제외
- TODO 제목 수정 전용 UI
- TODO 시간, 알림, 반복 일정
- 드래그 앤 드롭 일정 이동
- 공유 캘린더와 협업 기능

## 몽토리 메뉴
몽토리 메뉴는 사용자의 기록 상태를 요약하고, 앱의 동반자 성격을 보여주는 개인 대시보드다.

### 제공 기능
- `몽토리 컨디션` 카드에서 사용자, 데이터 소스, 오늘 일기/TODO 상태를 보여준다.
- `성장 기록` 카드에서 연속 기록, 누적 일기, TODO 완료, 감정 폭을 요약한다.
- `감정 팔레트` 카드에서 사용 가능한 감정 종류를 보여준다.
- `홈 위젯 미리보기` 카드에서 오늘 위젯에 보여줄 최소 정보를 미리 확인한다.

### 데이터 기준
- 사용자 정보는 로그인 세션에서 가져온다.
- 일기 통계는 현재 앱이 가진 일기 목록 기준으로 계산한다.
- TODO 통계는 현재 보이는 월의 TODO 목록 기준으로 계산한다.
- 감정 팔레트는 감정 목록 API 또는 mock datasource 결과를 사용한다.

### MVP에서 제외
- 캐릭터 성장 애니메이션
- 업적/배지 시스템
- 위젯 네이티브 렌더링
- 사용자 프로필 이미지 업로드

## 출품 관점 체크포인트
- 앱 진입 후 캘린더가 빈 보조 화면처럼 보이지 않아야 한다.
- 날짜 선택, 일기 작성, TODO 추가/완료가 한 화면에서 이어져야 한다.
- 몽토리 메뉴는 프로필 설정 화면이 아니라 현재 기록 상태를 이해하는 요약 화면으로 보여야 한다.
- 백엔드, Flutter mock, Flutter remote 모드에서 같은 개념 모델을 사용해야 한다.
